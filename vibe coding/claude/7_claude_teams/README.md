# 系统架构师辩论团队

## 概述

这是一个由 5 位不同专业领域的系统架构师组成的专家团队，他们将通过多轮辩论来设计最佳的后端系统架构。

## 工作流程

### Phase 1: 需求收集
每位架构师通过 `/brainstorming` 技能向用户询问项目需求，确保各自从专业角度理解项目。

### Phase 2: 独立设计
每位架构师基于需求，提出自己的架构设计方案。

### Phase 3: 多轮辩论
- **Round 1**: 互相挑战对方的设计假设
- **Round 2**: 针对争议点进行深度辩论
- **Round 3**: 寻找妥协方案和融合点

### Phase 4: 收敛决策
综合各方意见，形成最终的系统架构设计方案。

### Phase 5: 文档输出
生成完整的系统架构文档，包括：
- 架构图（ASCII/Markdown）
- 技术选型及理由
- 核心设计决策及权衡分析
- 各架构师的共识与分歧记录

## 使用方法

1. 启动后，每位架构师会依次向你提问项目需求
2. 回答后，每位架构师会提出自己的设计方案
3. 进入辩论环节，架构师们会互相挑战和论证
4. 最终输出一份综合性的系统架构文档
