
# ES6 中 `const`、`let` 和 `var` 的区别

## 1. `var`（函数作用域）

- **定义**：`var` 是 ES5 及更早版本的变量声明方式。
- **作用域**：`var` 是**函数作用域**的，这意味着在函数内部声明的 `var` 变量在整个函数中都是可见的。
- **变量提升**：`var` 具有变量提升（Hoisting）的特性，即在声明之前可以使用变量，但值为 `undefined`。
- **可重复声明**：在同一作用域中，可以用 `var` 重复声明相同的变量。

**示例**：
```javascript
function example() {
    console.log(x);  // 变量提升(提前声明,但不赋值) undefined
    var x = 10;
    console.log(x);  // 10
}
```

---

## 2. `let`（块作用域）

- **定义**：`let` 是 ES6 引入的，用于声明可变的变量。
- **作用域**：`let` 是**块作用域**的，意味着它只在最近的 `{}` 大括号中可见（如函数体、代码块、for 循环等）。
- **不允许重复声明**：在同一作用域中，不能重复声明同名的 `let` 变量。
- **变量提升**：`let` 也有变量提升，但它会产生一个**暂时性死区**（TDZ），在声明之前使用会抛出错误。

**示例**：
```javascript
function example() {
    console.log(x);  // ReferenceError: Cannot access 'x' before initialization
    let x = 10;
    console.log(x);  // 10
}

{
    let y = 5;
    console.log(y);  // 5
}
// console.log(y);  // ReferenceError: y is not defined
```

---

## 3. `const`（块作用域）

- **定义**：`const` 也是 ES6 引入的，用于声明常量，值一旦赋予便不能再改变。
- **作用域**：`const` 也是**块作用域**的，作用范围与 `let` 相同。
- **不可重新赋值**：`const` 声明的变量必须立即赋值，且之后不能更改它的值。
- **不允许重复声明**：`const` 在同一作用域中不能重复声明相同变量名。
- **变量提升与 TDZ**：`const` 也具有暂时性死区（TDZ），在声明之前使用会抛出错误。

**示例**：
```javascript
const x = 10;
x = 20;  // TypeError: Assignment to constant variable

const arr = [1, 2, 3];
arr.push(4);  // 允许修改引用类型的内容
console.log(arr);  // [1, 2, 3, 4]
```

---

## 4. 对比总结

| 特性             | `var`             | `let`            | `const`           |
|------------------|-------------------|------------------|-------------------|
| 作用域           | 函数作用域        | 块作用域         | 块作用域          |
| 变量提升         | 是                | 是（有暂时性死区）| 是（有暂时性死区）|
| 可重复声明       | 是                | 否               | 否                |
| 可重新赋值       | 是                | 是               | 否（但可修改对象属性） |
| 声明时必须赋值   | 否                | 否               | 是                |

---

## 5. 常见用法建议

- **`const`**：尽可能使用 `const`，因为它声明的变量不可改变，能减少意外的赋值错误。
- **`let`**：当你明确知道变量的值会改变时，使用 `let`。
- **`var`**：避免使用 `var`，因为其函数作用域和变量提升的特性容易导致不可预期的错误。
