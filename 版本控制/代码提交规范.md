# 代码提交规范

| 提交类型 | 描述                                                         |
| -------- | ------------------------------------------------------------ |
| feat     | 新功能（feature），表示从无到有，新增的功能                  |
| fix      | 修补bug                                                      |
| docs     | 文档（documentation）                                        |
| style    | 格式（不影响代码运行的改动）                                 |
| refactor | 重构（既不是新增功能，也不是修改bug的代码改动），或者增强逻辑功能 |
| test     | 增加测试，一般是单测和功能代码一起提交用feat，但若后面补一些单测，用这个 |
| chore    | 构建过程或辅助工具的变动，比如：pom、Dockfile等文件的改动    |
